# Система управління страховою компанією

Цей проект реалізує систему управління страховою компанією з багаторівневою архітектурою та функціоналом для різних ролей користувачів.

## Основні етапи реалізації

### Етап 1: Базові сутності та функціонал
- **Сутності**: Client, Policy, Agent, Claim.
- **Функціонал**: CRUD операції для клієнтів та полісів, створення нової страхової події, розрахунок вартості поліса, збереження даних у JSON/CSV/XML.

### Етап 2: Розширення бізнес-логіки та робота з подіями
- **Функціонал**: зміна статусів полісів та подій, пошук полісів за критеріями, автоматичне оновлення даних клієнта, завантаження даних з файлу.

### Етап 3: Статистика та аналітика
- **Функціонал**: розрахунок статистики (активні поліси, події за період, загальна сума виплат, дохід компанії, ефективність агентів), статистика по типах страхування, збереження запитів клієнтів.

### Етап 4: Нові сутності та розширення системи
- **Сутності**: Payment, Request.
- **Функціонал**: підбір полісів під запит клієнта, статистика по типах полісів.

### Етап 5: Перехід на базу даних та інтеграція
- **Функціонал**: перехід на базу даних через Entity Framework Core, інтеграція з віддаленим API.

### Етап 6: Ролі користувачів та поділ системи
- **Ролі**: клієнт, агент, менеджер.
- **Функціонал**: різні режими роботи для кожної ролі.

## Структура проекту

- **Models**: опис моделей даних.
- **Services**: бізнес-логіка системи.
- **Program.cs**: головний файл запуску програми.
